使用此命令与 WaaS 2.0 服务建立 WebSocket 连接并流式传输 webhook 事件，还可以选择将它们转发到指定的 URL。您可以使用 **Ctrl + C** 停止事件监听进程。

<Note>此命令主要用于开发和测试环境。对于生产环境，请按照 [设置回调或 webhook 端点](/v2_cn/guides/webhooks-callbacks/set-up-endpoint) 中的说明在您的 App 中设置适当的 webhook 端点。</Note>

## 用法

```shell
cobo webhook listen [options]
```

## 选项

- `--events <event_type>`：通过指定事件类型来过滤事件。将 `<event_type>` 替换为您感兴趣的特定事件类型，并用逗号分隔。事件类型的可能值包括：
    - `wallets.transaction.created`
    - `wallets.transaction.updated`
    - `wallets.transaction.failed`
    - `wallets.transaction.succeeded`
    - `wallets.mpc.tss_request.created`
    - `wallets.mpc.tss_request.updated`
    - `wallets.mpc.tss_request.failed`
    - `wallets.mpc.tss_request.succeeded`

    <Info>您也可以运行 [cobo webhook events](/v2_cn/developer-tools/cobo-cli/event-types) 命令来获取所有可用事件类型的最新列表。</Info>

- `--forward <url>`：将接收到的事件转发到指定的 URL。
    <Note>使用此选项时，请确保您的端点得到适当的安全保护，特别是在转发到公共 URL 时。</Note>

## 示例

```shell   
cobo webhook listen --events wallets.transaction.created,wallets.transaction.updated --forward "https://your-endpoint.com/webhook"
```

此命令监听 `wallets.transaction.created` 和 `wallets.transaction.updated` 事件类型，并将接收到的事件转发到 URL `https://your-endpoint.com/webhook`。
