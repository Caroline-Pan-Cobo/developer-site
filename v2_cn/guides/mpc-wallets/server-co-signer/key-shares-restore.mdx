恢复私钥分片意味着将备份的 TSS Node 数据移动到新主机并配置其以使用与以前相同的私钥分片。当您需要将 TSS Node 迁移到不同的服务器或从系统故障中恢复时，此过程是必要的。

您应在与原始部署相同的环境中恢复私钥分片。例如，如果您的 `secrets.db` 是在 Mac 上生成的，则应为恢复设置新的 Mac 环境。确保新主机符合[部署 TSS Node 的服务器要求](/v2_cn/guides/mpc-wallets/server-co-signer/tss-node-server-environments)中指定的服务器要求。

## 前提条件

在恢复私钥分片之前：
1. 按照[服务器要求](/v2_cn/guides/mpc-wallets/server-co-signer/tss-node-server-environments)在新主机上设置 TSS Node 环境
2. 下载用于备份的相同版本的 TSS Node 软件包

## 恢复过程

1. 解压 TSS Node 软件包
2. 将备份的 `db` 文件夹复制到解压目录中
3. 启动 TSS Node 

## 验证恢复

1. 检查您的私钥分片信息：
```bash
sudo ./tss-node.sh info group
```

2. 在输出日志中，验证根扩展公钥是否与 Cobo Portal 中显示的匹配。

## 完成恢复

1. 重启您的 TSS Node 
2. 输入创建备份时使用的加密密码
3. 如果密码正确，您的 TSS Node 将在新主机上正常运行
