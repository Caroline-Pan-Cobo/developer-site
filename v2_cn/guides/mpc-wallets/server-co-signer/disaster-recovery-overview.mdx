本指南提供了不同的恢复和灾难恢复方法概述，以处理服务端签名人的系统故障和灾难场景，并对它们进行比较。

## 可用方法

### 备份和恢复

为了防止系统故障或其他不可预见的情况，您应定期备份您的 TSS Node 数据。这包括：
- 包含密钥份额的 `secrets.db` 文件
- 数据库的加密密码

当您需要以下操作时使用恢复：
- 将您的 TSS Node 迁移到新主机
- 使用备份从系统故障中恢复

恢复过程使用您备份的数据来精确重建您的 TSS Node 环境。

有关详细信息，请参阅 [备份密钥份额](/v2_cn/guides/mpc-wallets/server-co-signer/key-shares-backup) 和 [恢复密钥份额](/v2_cn/guides/mpc-wallets/server-co-signer/key-shares-restore)。

### 恢复

根据您的情况，有两种恢复方法可用：

#### 从另一个活跃持有者组恢复密钥份额

在以下情况下使用此方法：
- 您的密钥份额丢失且无法访问备份
- 您有另一个活跃的密钥份额持有者组可用

此方法需要另一个活跃的密钥份额持有者组的协助来恢复您的密钥份额。

有关详细信息，请参阅 [从另一个活跃持有者组恢复密钥份额](https://manuals.cobo.com/en/portal/mpc-wallets/ocw/recover-from-active-keys)。

#### 恢复根扩展私钥
如果 Cobo Portal 停止服务，您可以利用活跃的恢复组重建与 Vault 对应的根扩展私钥。

有关详细信息，请参阅 [恢复根扩展私钥](https://manuals.cobo.com/en/portal/mpc-wallets/ocw/recover-root-key)。

## 方法比较

| 方法 | 何时使用 | 前提条件 | Cobo 参与 |
|--------|-------------|---------------|------------------|
| 备份和恢复 | 迁移到新主机或系统故障 | 备份数据 | 不需要 |
| 从另一个活跃持有者组恢复密钥份额| 丢失密钥份额且没有备份 | 活跃持有者组 | 需要 |
| 恢复根扩展私钥 | Cobo 服务不可用 | 恢复组| 不需要 |

## 最佳实践

1. **定期备份**
   - 定期备份您的 TSS Node 数据
   - 将备份存储在安全位置

2. **测试**
   - 定期测试您的恢复和恢复过程
   - 验证密钥功能在任何操作后
