This section provides an overview of the Cobo GraphQL API schema and types.

## Interface

### Address

Represents an address within a specified wallet.

**Fields**

| Field Name               | Type                     | Description                                                  |
| ------------------------ | ------------------------ | ------------------------------------------------------------ |
| `address`                | `String!`                | The wallet address.                               |
| `memo`                   | `String`                 | The memo code.                               |
| `path`                   | `String`                 | The derivation path of the address. This field applies to MPC Wallets only. To learn the meaning of each level in the path, see [Path levels](https://github.com/bitcoin/bips/blob/master/bip-0044.mediawiki#path-levels).                        |
| `encoding`               | `String`                 | The encoding format of the address. Available options are: `ENCODING_P2PKH`, `ENCODING_P2SH_P2WPKH`, `ENCODING_BECH32`, `ENCODING_P2PKH_UNCOMPRESSED`, `ENCODING_P2SH_P2MS`, `ENCODING_P2SH_P2WSH_P2MS`, and `ENCODING_P2TR`. |
| `pubkey`                 | `String`                 | The public key associated with the address. This field applies to MPC Wallets only.                     |
| `wallet`                 | [Wallet](/v2/developer-tools/graphql/schema#wallet)                 | The wallet that owns this address.                           |
| `chain`                  | [Chain](/v2/developer-tools/graphql/schema#chain)                  | The chain where the address exists.             |
| `balance`                | [TokenBalanceConnection](/v2/developer-tools/graphql/schema#tokenbalanceconnection) | The token balance for this address.                              |
| `taprootInternalAddress` | `String`                 | The Taproot address before tweaking.                         |
| `taprootScriptTreeHash`  | `String`                 | The hash of the script tree for Taproot transactions.        |
| `xOnlyPubkey`            | `String`                 | The 32-byte x-only public key in hexadecimal format after tweaking. |

**Implementations**

- [`CustodialWalletAddress`](#custodialwalletaddress)
- [`ExchangeWalletAddress`](#exchangewalletaddress)
- [`MPCWalletAddress`](#mpcwalletaddress)
- [`SmartContractWalletAddress`](#smartcontractwalletaddress)

### Asset

Represents an asset, including its metadata, associated coins, and balances.

**Fields**

| Field Name | Type | Description |
|----------|------|-------------|
| `code` | `String` | The asset ID. An asset ID is the unique identifier of the asset held within your linked exchange account. |
| `displayCode` | `String` | The asset symbol. You can use the value for display purposes. |
| `iconUrl` | `String` | The URL of the asset icon. |
| `coins` | [`[Coin]`](#coin) | A list of supported coins under this asset. |
| `totalBalance` | `Decimal` | The total balance of the asset. |
| `totalValue` | `Decimal` | The total value of the asset. |
| `totalAmount` | `Decimal` | The total amount of this asset held. |
| `lastActivityTimestamp` | `BigInt` | The timestamp (in Unix format) of the last recorded activity related to this asset. |
| `description` | `String` | A description of the asset. |


### Balance

Represents different types of balances for a token.

**Fields**

| Field Name  | Type     | Description                                     |
| ----------- | -------- | ----------------------------------------------- |
| `total`     | `String` | The current amount of tokens in an address, which is retrieved directly from the network. To learn more, see [Balances and transaction amounts for MPC Wallets](/v2/guides/mpc-wallets/balance-amounts).         |
| `available` | `String` | The amount of tokens ready to be spent. To learn more, see [Balances and transaction amounts for MPC Wallets](/v2/guides/mpc-wallets/balance-amounts).                        |
| `pending`   | `String` | The total amount being sent in a transaction, which is calculated as the withdrawal amount plus the transaction fee. To learn more, see [Balances and transaction amounts for MPC Wallets](/v2/guides/mpc-wallets/balance-amounts).       |
| `locked`    | `String` | For UTXO chains, this is the combined value of the selected UTXOs for the transaction. For other chains, it is equal to the Pending amount. To learn more, see [Balances and transaction amounts for MPC Wallets](/v2/guides/mpc-wallets/balance-amounts). |

### CalldataInfoType

Represents the calldata information for a contract.

**Fields**

| Field Name | Type | Description |
|----------|------|-------------|
| `address` | `String` | The address of the smart contract. |
| `chainId` | `String` | The chain ID where the contract is deployed. |
| `contractName` | `String` | The name of the contract. |
| `impl` | `String` | The address of the implementation smart contract. This field is applicable only when the specified smart contract is a proxy contract. |
| `implName` | `String` | The name of the implementation smart contract. This field is applicable only when the specified smart contract is a proxy contract. |
| `proxy` | `Boolean` | Indicates whether the contract is a proxy. |
| `data` | `String` | The raw calldata as a hex string. |
| `method` | [`ContractMethodType`](#contractmethodtype) | The information about a method in a smart contract. |
| `params` | `String` | The parameters of the contract method are represented as a JSON array of arrays. Each element in the outer array is itself an array containing three elements that provide detailed information about a specific parameter:<ul><li>Parameter name: The unique identifier of the parameter, such as kind, swaps, and to.</li><li>Parameter type: The Solidity data type of the parameter, such as uint8, tuple\[\], address\[\], and int256\[\].</li><li>Parameter value: The actual value of the parameter. If the parameter type is a basic type such as uint256 or address, this value is a single element. If the parameter type is a complex type such as tuple\[\] or address\[\], the value is a nested array, with each inner array containing parameter names, types, and values.</li></ul> |

### ContractMethodType

Represents a method in a smart contract.

**Fields**

| Field Name | Type | Description |
|----------|------|-------------|

| `sig` | `String` | The signature of the method, which includes the method name and parameter types. |
